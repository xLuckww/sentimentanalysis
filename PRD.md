# 产品需求文档（PRD）—— 中文评论分析工具

## 1. 项目背景与目标

### 1.1 背景
在做用户调研、产品反馈分析、电商评论分析等场景时，经常需要对大量中文文本评论进行词频统计、可视化和情感倾向判断。市面上的工具要么是付费 SaaS，要么是面向程序员的 Jupyter notebook 脚本，缺少一个**开箱即用、无需写代码、面向非技术用户**的桌面端工具。

### 1.2 目标
开发一个 macOS 桌面应用，让用户能够：
1. 上传一份评论数据集（CSV/Excel）
2. 指定评论文本所在的列
3. 一键完成中文分词、停用词过滤、词频统计、词云生成、情感分析
4. 直观地查看和导出分析结果

### 1.3 项目定位
本项目兼具两个目标：
- **个人作品集 / 求职项目**：作为产品经理（PM）方向求职的项目作品，需要体现完整的产品思维（需求定义、功能设计、迭代规划），代码与产品文档需具备可展示性。
- **实际可用工具**：功能本身要真实可用，能处理真实的中文评论数据集（如电商评论、问卷开放题、社交媒体评论等），不是一个 demo 级别的玩具项目。

### 1.4 目标用户
- 需要做用户反馈/评论分析的产品经理、运营、市场人员
- 没有编程基础，但需要从文本数据中提炼洞察的非技术用户
- 求职作品集场景下，会浏览该项目的招聘方/面试官

## 2. 技术方案概览

| 项目 | 选型 | 备注 |
|---|---|---|
| 开发语言 | Python 3.10+ | |
| GUI 框架 | PySide6 (Qt for Python) | 接近原生 macOS 体验，可打包为独立 .app |
| 数据处理 | pandas | 读取 CSV/Excel，列操作 |
| 中文分词 | jieba | 支持自定义词典 |
| 停用词过滤 | 内置词表 + 用户自定义 | |
| 词频可视化 | matplotlib | 条形图 |
| 词云生成 | wordcloud | 需配置中文字体 |
| 情感分析 | SnowNLP | 词典法，离线运行，无需 API |
| 打包 | py2app 或 PyInstaller | 输出独立 .app 文件 |

> 注：情感分析当前版本使用 SnowNLP（离线、免费、准确率中等）。架构设计上需预留情感分析模块的可替换接口，便于未来切换/接入更高准确率的方案（如大模型 API），但**本版本不实现**该切换功能，仅做接口预留。

## 3. 功能需求

### 3.1 整体交互流程
采用**向导式（Wizard）四步流程**，用户按顺序完成：

```
第一步：上传数据 → 第二步：选择评论列 → 第三步：分析配置 → 第四步：结果展示
```

建议使用左侧步骤导航栏（Stepper）+ 右侧内容区的布局，已完成步骤可回退修改，未完成步骤置灰不可跳转。

---

### 3.2 第一步：上传数据

**功能描述：**
- 支持拖拽文件到窗口，或点击按钮打开文件选择对话框
- 支持文件格式：`.csv`、`.xlsx`、`.xls`
- CSV 文件需自动检测编码（优先尝试 UTF-8，失败则尝试 GBK/GB2312，中文评论数据集很多是 GBK 编码导出的，这一步如果处理不好会直接导致乱码，是高优先级细节）
- 上传成功后，立即在界面展示数据预览表格（前 20 行）
- 展示基本信息：总行数、总列数、文件大小

**异常处理：**
- 文件格式不支持 → 明确提示用户支持的格式
- 文件为空或解析失败 → 提示具体错误原因（不能只说"上传失败"）
- 文件过大（建议设置 50MB 或 10 万行的软上限提示，超出时提醒用户可能分析较慢，但不强制阻止）

---

### 3.3 第二步：选择评论所在列

**功能描述：**
- 以下拉框形式列出第一步上传数据的所有列名
- 选中某一列后，下方预览区展示该列的前 10 条内容，供用户确认选对了列
- 自动检测可能是文本列的候选项并置顶推荐（例如：列名包含"评论""内容""comment""review""反馈"等关键词，或该列平均字符长度明显长于其他列）
- 显示该列的非空值数量、空值数量（让用户了解数据完整度）

**异常处理：**
- 选中列全部为空值 → 提示用户该列无有效数据，建议重新选择
- 选中列内容明显不是文本（如全是数字/日期）→ 给出提示但不强制阻止（用户可能就是要分析数字列的边界情况，不做过度假设）

---

### 3.4 第三步：分析配置

**功能描述（可配置项）：**

| 配置项 | 说明 | 默认值 |
|---|---|---|
| 停用词表 | 内置通用中文停用词表，同时支持用户上传自定义停用词表（.txt，一行一词）或在界面手动添加/删除词语 | 使用内置词表 |
| 自定义分词词典 | 支持上传 jieba 格式自定义词典（用于避免产品名、专有名词被错误切分），格式说明需在界面标注或提供示例文件下载 | 不启用 |
| 过滤单字词 | 是否过滤掉分词结果中的单字词（如"的""是"等残留单字，通常意义不大） | 开启 |
| 过滤数字 | 是否过滤纯数字词 | 开启 |
| 词频统计 Top N | 条形图与词云展示的高频词数量 | Top 30（可调范围 10-100） |
| 启用情感分析 | 是否对每条评论进行情感打分 | 开启 |

**预处理细节（需在开发时内置，非用户可见配置）：**
- 分词前需做正则清洗：去除 URL、@用户名、表情符号（如 `[微笑]` 这类文本表情）、多余空白符
- 标点符号统一过滤，不计入词频统计

**交互要求：**
- 配置项变更后提供"开始分析"按钮，点击后进入处理中状态（显示进度条或加载动画，避免用户以为程序卡死，尤其是数据量大时分词和情感分析都需要时间）
- 分析过程在后台线程执行，不能阻塞 UI 主线程（PySide6 中需用 QThread 或 QRunnable 处理，避免界面卡死无响应）

---

### 3.5 第四步：结果展示

**功能模块：**

#### A. 词频统计 - 条形图
- 横向或纵向条形图展示 Top N 高频词及对应频次
- 支持鼠标悬停查看精确数值
- 支持导出为图片（PNG）

#### B. 词频统计 - 词云图
- 标准词云图，词语大小对应频次
- 需正确显示中文（提前配置好中文字体路径，这是最容易踩坑的地方）
- 支持导出为图片（PNG）
- 支持调整配色方案（提供 2-3 种预设配色供选择即可，不需要做成完全自定义调色板）

#### C. 情感分析结果
- 整体情感分布：正面 / 中性 / 负面 占比，用饼图或环形图展示
  - SnowNLP 输出的是 0-1 的情感得分（越接近1越正面），需要自行定义分档阈值，建议默认：> 0.6 为正面，< 0.4 为负面，中间为中性，此阈值在 PRD 阶段先写死，不做用户可调（除非后续迭代）
- **情感明细列表**：允许用户按情感类别筛选（点击"负面"标签后，下方表格展示所有被判定为负面的原始评论及对应得分），这一点对实际分析价值很大，优先级高于美观的图表
- 表格支持按情感得分排序

#### D. 结果导出
- 支持将完整分析结果导出为 Excel 文件，至少包含以下几个 sheet：
  - 原始数据 + 新增"情感得分""情感分类"两列
  - 词频统计表（词语、频次）
- 支持单独导出条形图、词云图为 PNG
- 导出路径默认桌面，但允许用户选择保存位置

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 1万条评论的完整分析（分词+情感分析+可视化）应在 30 秒内完成，超过该时长需有明确进度提示 |
| 稳定性 | 任何步骤出错都不应导致程序崩溃，需有 try-except 包裹并给出用户可理解的错误提示，而非显示 Python 报错堆栈 |
| 兼容性 | 支持 macOS 12 (Monterey) 及以上版本 |
| 中文支持 | 全流程中文界面，所有图表、字体、编码均需正确处理中文，不出现乱码或方块字 |
| 数据隐私 | 所有数据处理均在本地完成，不上传任何数据到外部服务器（情感分析使用离线 SnowNLP，符合这一原则） |

## 5. 版本规划

### V1.0（MVP，本次开发目标）
- 完整实现第 3.2 - 3.5 节所述的全部功能
- 打包为可双击运行的 macOS .app

### V2.0（后续迭代方向，本次不开发，仅做架构预留）
- 情感分析支持切换为大模型 API 模式（更高准确率）
- 支持多列对比分析（如同时分析"标题"和"正文"两列）
- 支持中文专属可视化扩展，如关键词共现网络图
- 历史分析记录管理（保存多次分析结果供回顾对比）

## 6. 验收标准

1. 用户可以成功上传一份 GBK 编码的 CSV 文件并正确显示中文，无乱码
2. 选择评论列后能看到准确的预览内容
3. 调整停用词配置后重新分析，结果能正确反映配置变化（如关闭"过滤单字词"后单字词重新出现在结果中）
4. 词云图中文显示正常，无方块/乱码
5. 情感分析结果的正负面判定与人工抽样判断大致一致（允许词典法本身的误差，不要求100%准确）
6. 点击"负面"筛选后，列表中展示的评论确实是负面倾向的内容
7. 导出的 Excel 文件可以正常用 Excel/Numbers 打开，内容完整
8. 在没有 PySide6 开发环境的电脑上，双击打包好的 .app 文件可以直接运行
